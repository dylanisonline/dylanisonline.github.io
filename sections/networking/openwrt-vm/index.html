OpenWRT Virtual Machine Setup


OpenWrt acts as the lab’s virtual router and network gateway. It provides routing, firewall control, DHCP services, and traffic separation between the server and workstation subnets, helping the environment behave more like a real small enterprise network.


Preparing OpenWRT .vmdk

By default, OpenWRT is downloaded as a .img file since it meant to be flashed onto network hardware. Since VMware Workstation does not natively support .img files, I needed to convert it to a .vmdk file.

After some research, I decided to using qemu-tools in WSL 2.0 in order to complete the conversion.


Installing Qemu Tools on Ubuntu WSL 2.0


> sudo apt update
> sudo apt install qemu-utils -y

### Creating Directory in my home folder for the files
> mkdir openwrt


Converting .img to .vmdk


> qemu-img convert -f raw -O vmdk openwrt.img openwrt.vmdk
> ls

### openwrt.img openwrt.vmdk


VMware Workstation Settings

To prepare the environment for OpenWrt, I first configured VMware’s virtual networks to act as the backbone of the lab. These networks allow the router, servers, and workstations to communicate while keeping each subnet properly separated.


VMware Virtual Networks


AD-SERVERNET — 10.10.10.0/24

  • Host-only VMware network
  • Used by domain controllers, file servers, and other infrastructure servers
  • VMware DHCP disabled
  • IP addressing and network services are managed through OpenWrt

AD-WORKSTATION — 10.10.20.0/24

  • Host-only VMware network
  • Used by domain-joined workstations and client devices
  • VMware DHCP disabled
  • OpenWrt provides DHCP leases and routing for connected workstations

WAN

  • Connected to VMware’s NAT-enabled network
  • DHCP enabled
  • Allows the OpenWrt WAN interface to automatically receive an IP address
  • Provides internet access to the internal server and workstation networks through OpenWrt



OpenWRT Virtual Machine Settings



Hard Disk: With the converted .vmdk file, we can attach the boot drive to the virtual hard disk image to boot into Openwrt.

Network Adapter: Connected to VMware’s NAT-enabled network. This interface receives an IP address through DHCP and provides the OpenWrt router with internet access.

Network Adapter 2: Connected to the AD-SERVERNET host-only network. This interface serves as the default gateway for domain controllers, file servers, and other infrastructure servers on the 10.10.10.0/24 subnet.

Network Adapter 3: Connected to the AD-WORKSTATION host-only network. This interface provides routing and DHCP services for domain-joined workstations on the 10.10.20.0/24 subnet.
